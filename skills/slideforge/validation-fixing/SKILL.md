# SlideForge Validation & Layout Auditing

This leaf skill guides parameter validation and visual layout auditing. SlideForge includes a built-in validation engine designed to catch design errors before you render or export.

## Pre-flight Parameter Validation

Run `validate-layout` (CLI) or `validate_layout` (MCP) before generating slides to ensure the input data matches structural expectations.
- **Goal:** Confirms required parameters are present, array bounds are correct, and types match.

```bash
# Validate slide parameters prior to generation
slideforge-rust validate-layout hero --params '{"headline":"My Title"}'
```

---

## Rendered HTML Design Validation

After compiling the carousel HTML but before running exports, run `validate-design` (CLI) or `validate_design` (MCP).
- **Goal:** Parsers inspect the rendered DOM structure for:
  1. **Overflow:** Elements breaking the boundaries of the slide viewport.
  2. **Clipping:** Tight line heights or font offsets cutting off descender characters (g, j, p, q, y).
  3. **Contrast:** Insufficient color contrast between foreground text and backgrounds (min 4.5:1 ratio).
  4. **Squishing:** Text columns wrapped so narrow they become unreadable, or card columns forced to wrap.

```bash
# Audit design output for visual defects
slideforge-rust validate-design ./carousel.html
```

## Common Validation Failures & Fixes

When the validator flags an issue, apply these specific repairs:

### 1. "Text clipping risk" / "Descender overflow"
- **Cause:** Headline or quote text is too long for its container size, forcing CSS overflow wrappers to crop the text.
- **Repair:** Shorten the text parameter string, or insert manual `\n` breaks to divide the line. Avoid decreasing line height.

### 2. "Column width constricted"
- **Cause:** Too many columns used in `text_columns`, `grid_cards`, or `split_features`.
- **Repair:** Limit `text_columns` to 2-3, `grid_cards` to 3 items, and `split_features` to 2.

### 3. "Contrast check failed"
- **Cause:** Using custom overrides (`--override text_color=...`) that blend with background gradients.
- **Repair:** Revert to the default archetype color mappings generated by `configure-design`, which are derived using perceptual science (OKLCH).

## Actionable Verification Checklist

- [ ] **No Warnings:** Never ignore validation output. Treat warnings as errors.
- [ ] **Check HTML Locally:** View the compiled HTML inside a browser before exporting.
- [ ] **Fix and Re-Validate:** Run validation again after making any content adjustments.
